# Стек: kie.ai (генеративные изображения / видео / аудио)

- Auth: `Authorization: Bearer <key>`, `Content-Type: application/json`.
  Ключ — в настройках/БД (переопределяемо через админку), в чат/логи не выводить.
- Создание задачи: `POST {base}/api/v1/jobs/createTask` (у отдельных моделей свои
  пути, например Veo: `/api/v1/veo/generate`).
- **ГЛАВНАЯ ЛОВУШКА**: kie.ai отдаёт HTTP 200 с ошибкой в теле — всегда проверяй
  `body["code"]`: `code != 200` → это ошибка, поднимай исключение.
- Результат: либо вебхук (в payload инжектится `callBackUrl`), либо поллинг
  `GET /api/v1/jobs/recordInfo?taskId=...` — статус в `data.state`
  (`success`/`failed`); интервалы поллинга подбирать пер-модель.
- Билдеры payload — по модели (dict-роутер `имя → _build_*`);
  httpx-таймауты на каждый вызов.
- Тяжёлые генерации — в фоновые задачи (Celery), не в обработчике запроса.
- Приоритетные модели для фото: nano banana 2, gpt image 2.
