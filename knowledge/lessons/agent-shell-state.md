# Состояние шелла в агентных Bash-вызовах не персистится

Стек: любой агентный CLI-харнесс (Claude Code и подобные), где команды выполняются отдельными Bash-вызовами.

## Симптомы
- «command not found» для базовых утилит (curl, wc) во втором и последующих вызовах, хотя первый работал — PATH урезан или сброшен.
- Файлы, скачанные curl или созданные скриптом, оказываются в корне репозитория вместо рабочей папки — cwd сброшен между вызовами, `cd` из предыдущего вызова не действует.

## Правило
- Каждую команду начинать с `export PATH=/usr/bin:/bin` (при необходимости добавить /opt/homebrew/bin и т.п.) либо звать бинари по абсолютным путям.
- Все пути файлов — только абсолютные; на `cd` предыдущего вызова не полагаться. Если внутри команды нужен cd — делать его в этой же команде.
- Самопроверка: если команда пишет файлы — сразу `ls` по абсолютному пути назначения; «тихое» попадание файлов в корень репо ломает изоляцию проектов.

## Цена ошибки
Дважды за одну сессию файлы генерации падали в корень репозитория (нарушение изоляции клиентских папок), плюс ложный 401 из-за относительного пути к файлу с ключом.
