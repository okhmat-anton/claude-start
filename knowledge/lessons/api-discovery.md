# API discovery — когда живой API опережает документацию

## Недостающие эндпоинты/поля не угадывать

Контекст: проекты, где данные идут через внешний REST API, а доступ описан локальным доком (AGENTS-файл, playbook). Документация регулярно отстаёт от живого API.

Урок — порядок поиска, когда нужного эндпоинта или поля нет в основном доке:
1. Синхронизировать вторичные доки (playbook методологии) по хэшу версии и искать там.
2. Проверить журнал уроков проекта (LESSONS.md) — эндпоинт мог быть уже найден и проверен раньше.
3. Посмотреть shape живого GET-ответа: новые поля появляются в API раньше, чем в доках; смежные поля (`*_id`, счётчики в `/summary`) подсказывают существование эндпоинтов.
4. Запись проверять пробным PUT/POST на ОДНОМ объекте с проверкой echo в ответе — и только потом батч.

Ничего не выдумывать по памяти и не писать «осторожных хеджей» в инструкции про эндпоинты, чей статус уже установлен, — сверяться с журналами.
